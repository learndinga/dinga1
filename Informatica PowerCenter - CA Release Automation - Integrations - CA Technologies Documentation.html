                  Informatica PowerCenter

                  Last update January 30, 2017

                  Action Pack for Informatica PowerCenter 10.1.1502

                  The Action Pack for Informatica PowerCenter contains actions that enable you to create workflow processes that perform operations in Informatica PowerCenter.

                  • 1Action Pack for Informatica PowerCenter 10.1.1502
                  • 2What's New
                  • 3Install the Action Pack
                  • 4Prerequisites to Run Informatica PowerCenter Actions
                  • 5Informatica PowerCenter Actions

                  Actions Help

                  For a list of input and output descriptions for deployed actions, refer to:

                  • Informatica PowerCenter Actions in this document
                  • The embedded actions help that is available in your installation at http://<yourdeployment>/nolio-app/actionshelp.jsp.

                  Supported Platforms:

                  • Informatica PowerCenter actions in this pack support Informatica PowerCenter 9.0 and later
                  • Informatica PowerCenter actions in this pack support CA Release Automation 5.0 and later
                  • Windows server enterprise (32 bit)
                  • Linux Server 5.5 (64 bit)

                  What's New

                  The following update was made for 10.1.1502

                  • Removed the json-path-*.jar dependency file.

                  Install the Action Pack

                  Install and use these actions to execute commands and tasks in Informatica PowerCenter.

                  Follow these steps:

                  1. Verify the Prerequisites to Run Informatica PowerCenter Actions.
                  2. Deploy the action pack for Informatica PowerCenter.
                    For more information, see Manage Action Packs and Plug-ins.
                  3. Use Informatica PowerCenter actions in CA Release Automation processes.

                  Prerequisites to Run Informatica PowerCenter Actions

                  To run Informatica PowerCenter actions, verify the following prerequisites:

                  1. The Agent is required to be installed on the PowerCenter server to use PowerCenter actions.
                  2. The PMREP command-line tool is installed on the PowerCenter Client and in the PowerCenter Services bin directories.

                  3. If manually copying and importing the PowerCenter Action pack, also import the RDKCommon-1.0.jar file.

                    Note: The working directory of these actions is: c:\ for Windows and / for Linux. If you use the Run Command-line action with PMREP, combined with this action pack, set c:\ or / in the Working Directory field.

                  Informatica PowerCenter Actions

                  The Action Pack for Informatica PowerCenter provides the following actions:

                  • 1IPC - Add Object to Deployment Group
                  • 2IPC - Create Deployment Group
                  • 3IPC - Register Local Repository
                  • 4IPC - Roll Back Deployment

                  IPC - Add Object to Deployment Group

                  Adds an object to a deployment group.

                  Name Type Description
                  * Deployment group name String Name of the deployment group to add objects to.
                  * PMREP path String The directory where the PMREP executable is located.
                  Dbd separator String (Optional). If an ODBC source has a period (.) in the name, define a different separator character when you define the source object. For example, instead of database_name.source_name, define the source object as database_name\source_name, and define the dbd_separator as backslash (\).
                  Dependency types String Dependent objects to add to the deployment group with the object. Enter one of the following: all. pmrep adds the objects and all dependent objects, reusable and non-reusable, to the deployment group. “non-reusable”. pmrep adds the objects and the corresponding non-reusable dependent objects to the deployment group. Note: pmrep does not add dependent objects to the deployment group. If you omit this parameter, pmrep adds the objects and all dependent objects to the deployment group.
                  Folder name String Folder that contains the object you are adding.
                  Object name String Name of the object you are adding to the deployment group. You cannot enter the name of a checked out object.
                  Object subtype String Type of task or transformation you are adding.
                  Object type String Type of object you are adding. You can specify the source, target, transformation, mapping, session, worklet, workflow, scheduler, session configuration, task, cube, and dimension.
                  Output File Location String Specifies the location and file name where the output of the command is written. For example, C:\temp\output.txt
                  Persistent input file String A text file that is generated from ExecuteQuery, Validate, or ListObjectDependencies that contains a list of object records with encoded IDs. If you use this parameter, do not input Object name, Object type, and Folder name.
                  Version number String Version of the object to add. The default is the latest version of the object. The command fails if you specify a version number for a non-versioned repository.
                  Name Type Description
                  Error Output String This output parameter holds the standard error output of the command execution.
                  Execution Output String This output parameter holds the standard output of the command execution.
                  Exit Code Integer This output parameter holds the returned exit code of the command execution.

                  IPC - Create Deployment Group

                  Creates a deployment group.

                  Name Type Description
                  * Deployment group name String Name of the deployment group to create.
                  * PMREP path String The directory where the PMREP executable is located.
                  Comments String (Optional) Comments about the new deployment group.
                  Deployment group type String Optional. Create a static group or use a query to dynamically create the group. You can specify static or dynamic. Default is static.
                  Output File Location String Specifies the location and filename where the output of the command is written. For example, C:\temp\output.txt
                  Query name String Required if the deployment group is dynamic, is ignored if the group is static. Name of the query that is associated with the deployment group.
                  Query type CreateDeploymentGroup$querytype6Type Required if the deployment group is dynamic. Ignored if the group is static. Type of query to create a deployment group. You can specify shared or personal.
                  Name Type Description
                  Error Output String This output parameter holds the standard error output of the command execution.
                  Execution Output String This output parameter holds the standard output of the command execution.
                  Exit Code Integer This output parameter holds the returned exit code of the command execution.

                  IPC - Register Local Repository

                  Registers a local repository with a connected global repository.

                  Machine name of the domain where the local repository is located. Required if the local repository is in a different domain and you do not use Local repository domain name. If you use this option, input the Local repository portal port number field.Port number for the domain where the local repository is located. Required if the local repository is in a different domain and you do not input Local repository domain name. If you input this field, input the Local repository portal host name field.

                  Name Type Description
                  * Local repository name String Name of the local repository to register.
                  * Local repository password String Log in password for the local target repository.
                  * Local repository user name String Local user name.
                  * PMREP Path String The directory where the PMREP executable is located.
                  Local repository domain name String Name of the Informatica domain for the repository. Required if the local repository is in a different domain and you do not input the Local repository portal host name and Local repository portal port number fields.
                  Local repository portal host name String
                  Local repository portal port number Integer
                  Local repository user security domain String Required if you use LDAP authentication. Name of the security domain that the user belongs to. Default is Native.
                  Output File Location String Specifies the location and filename to which the output of the command is written. For example, C:\temp\output.txt
                  Name Type Description
                  Error Output String This output parameter holds the standard error output of the command execution.
                  Execution Output String This output parameter holds the standard error output of the command execution.
                  Exit Code Integer This output parameter holds the returned exit code of the command execution.

                  IPC - Roll Back Deployment

                  Rolls back a deployment to purge a deployed version from the target repository or folder and revert objects to a previous version of a deployment.

                  Name Type Description
                  * Deployment group name String Name of the deployment group to roll back.
                  * Nth latest deploy path String Version of the deployment you want to roll back.
                  * PMREP path String The directory where the PMREP executable is located.
                  Output File Location String Specifies the location and file name where the output of the command is written. For example, C:\temp\output.txt
                  Name Type Description
                  Error Output String This output parameter holds the standard error output of the command execution.
                  Execution Output String This output parameter holds the standard output of the command execution.
                  Exit Code Integer This output parameter holds the returned exit code of the command execution.
