Informatica PowerCenter Continuous Delivery solution

Informatica PowerCenter Continuous Delivery solution offered by Hopsas is a set of Jenkins (https://jenkins.io/index.html) jobs, that allow for building, testing, and releasing Informatica ® code faster and more frequently.

These jobs mainly wrap around original „pmrep” command-line utility offered by Informatica ®.

There are many advantages of using dedicated GUI over original command-line interface. Some of these include:

  • All jobs executions are recorded, its output persisted in Jenkins database for further analysis / evidence.
  • All jobs executions can be easily repeated, if necessary.
  • Single job instance can be used to execute same tasks (including deployment) across every environment (System Test, User Acceptance Tests, Maintenance, Production etc.) Code deployment method (using deployment groups) is therefore standardized / unified and much faster than manual tasks.
  • Jenkins scripts can be easily modified to i.e.: execute deployments of deployment groups with control / parameter files taken from external repositories, such as SVN, Git, Mercurial etc.
  • Selected jobs i.e.: Informatica ® repository database optimization (reindexing) can be executed on a scheduled basis directly by Jenkins itself.
  • No need to memorize syntax of every „pmrep” command – everything can be selected from drop-lists or manually entered into required field. Also, users with no technical knowledge / no Informatica ® client installed can execute predefined jobs on code repositories.

Jenkins jobs offered by current version of our Continuous Delivery solution are listed below:

They can be divided into two logical parts:

  • Jobs prefixed INFA_CDELIVERY_DEV_ are used mainly by Informatica ® developers and administrators.
  • Jobs prefixed INFA_CDELIVERY_ADMIN_ are used mainly by Informatica ® administrators.

Specific job purpose / description is as follows:

INFA_CDELIVERY_ADMIN_UPDATE_STATISTICS

This job updates statistics for indexes created on Informatica Repository (relational database). This allows for faster repository data access. This job should be executed at least once a month by Informatica Administrator (it is possible to schedule automatic job execution inside Jenkins).

 

INFA_CDELIVERY_ADMIN_TRUNCATE_LOGS

This job truncates logs persisted in Informatica repository that are older than ‚x’ days. This job should be executed at least once a month by Informatica Administrator (it is possible to schedule automatic job execution inside Jenkins).

 

INFA_CDELIVERY_ADMIN_NOTIFY_USERS

This job notifies Informatica developers with admin defined message. It`s very convenient as message sent by Informatica administrator is clearly visible inside every developer tool connected to specified Informatica repository.

 

INFA_CDELIVERY_ADMIN_CREATE_FOLDER

This job creates new folder in selected Informatica repository. Global / shared folders are also supported.

 

INFA_CDELIVERY_ADMIN_DEPLOY_DEPLOYMENT_GROUP

This job deploys a deployment group into specified target repository by deployment group name. Uses deployment control file created by Informatica developer.

 

INFA_CDELIVERY_DEV_LIST_OBJECTS

This job lists objects defined in Informatica repository by type and folder.

 

INFA_CDELIVERY_DEV_LIST_CONNECTIONS

This job lists all connections to external sources / targets defined in Informatica repository.

 

INFA_CDELIVERY_DEV_GET_CONNECTION_DETAILS

This job gets details of Informatica connection specified by name and type. List of all connections may be retrieved by another Jenkins job.

 

INFA_CDELIVERY_DEV_FIND_CHECKOUTS

This job finds all checkouts in Informatica repository.

 

INFA_CDELIVERY_DEV_LIST_OBJECT_DEPENDENCIES

This job lists specific object dependencies (children, parents, or both) across Informatica repository.

 

INFA_CDELIVERY_DEV_LIST_TABLES_BY_SESSION

This job lists sources and / or targets connected with specified Informatica session.

 

INFA_CDELIVERY_ADMIN_ROLLBACK_DEPLOYMENT_GROUP

This job undeploys (rollbacks) previously deployed deployment group.
